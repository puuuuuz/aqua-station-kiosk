# 📡 แผนการอัปเดตระบบ Kiosk ทางไกล (OTA Architecture Strategy)

สำหรับการบริหารจัดการตู้จำนวนมากโดยไม่ต้องเดินทางไปที่หน้างาน (Zero-Visit Maintenance) เราจะใช้กลยุทธ์แบบ 2 ระดับครับ:

## 1. การอัปเดต UI และ Logic (Web OTA) - **[ใช้งานได้แล้วปัจจุบัน]**
เนื่องจากหัวใจของตู้ (UI, สต็อกน้ำ, การคำนวณราคา, ระบบ Serial) ทั้งหมดรันอยู่บน **Vercel** ผ่านระบบ **WebView**:
*   **วิธีการ:** ทันทีที่ผม (Antigravity) หรือพี่ ทำการแก้ไขโค้ดและ `vercel deploy` ระบบทุกตู้จะได้รับโค้ดใหม่ทันที
*   **การอัปเดต:** พี่แค่สั่งรีเฟรชหน้าจอ (ผ่านปุ่ม Reset หรือส่งคำสั่ง Remote Refresh) ตู้ก็จะโหลดหน้าเว็บเวอร์ชันล่าสุดมาใช้งานทันทีโดยไม่ต้องลง APK ใหม่ครับ

## 2. การอัปเดต APK (Native OTA) - **[แผนงานสำหรับอนาคต]**
หากในอนาคตมีการแก้ไขสิ่งที่เกี่ยวกับฮาร์ดแวร์ลึกๆ (เช่น เปลี่ยนฟังก์ชัน Bridge, เปลี่ยน Permission Android):
*   **ระบบ App Auto-Update:** เราจะเพิ่มโค้ดใน Android (Java/Kotlin) ให้ตรวจสอบเวอร์ชันจาก URL กลาง (เช่น JSON file) ทุกครั้งที่เปิดแอป
*   **Silent Update (Root Required):** ถ้าเครื่องถูก Root ไว้ (ตู้ส่วนใหญ่มักจะ Root มา) เราสามารถเขียนสคริปต์ให้แอปทำการ `pm install -r` (ติดตั้งทับ) และรีสตาร์ทตัวเองแบบเงียบๆ ได้เลยครับ
*   **Update Prompt (Non-Root):** ถ้าไม่ Root จะมีหน้าต่างเด้งขึ้นมาให้กด "ตกลง" เพื่ออัปเดตครับ

## 3. การควบคุมทางไกล (Remote Control)
เพื่อความสะดวกในการดูแลเครื่องจำนวนมาก ผมแนะนำให้ติดตั้ง **AnyDesk** หรือ **TeamViewer (Host)** ไว้ที่ตู้ด้วยครับ:
*   ส่องหน้าจอเพื่อดูปัญหาจริงที่หน้าตู้
*   กด Reboot หรือ Clear Cache ได้จากมือถือหรือคอมพิวเตอร์ที่บ้าน

---

> [!TIP]
> **สรุป:** ตอนนี้พี่มีระบบ Web-OTA อยู่ในมือแล้วครับ ทุกอย่างที่ผมแก้ให้พี่วันนี้ (เรื่อง Flow, เรื่อง Save) พี่ไม่ต้องไปที่ตู้เลยครับ แค่กดรีเฟรชที่หน้าจอตู้ ข้อมูลใหม่จะถูกโหลดมาใช้งานทันทีครับ!
