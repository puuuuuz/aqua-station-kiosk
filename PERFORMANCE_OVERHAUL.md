# 🏎️ แผนผังการจูนความพริ้วระดับ 100% (Performance Overhaul Plan)

พี่ครับ หลังจากที่ผมวิเคราะห์อาการ "ค้าง/กระตุก" อย่างละเอียด ผมพบว่ามันไม่ได้สาเหตุมาจาก Polling อย่างเดียว แต่เป็นที่ **"Layout Thrashing" (ภาระหน้าจอ)** ครับ นี่คือแผนการจัดระเบียบใหม่ทั้งหมดครับ:

## 1. 🛑 ปัญหาสามเหลี่ยม (The Bottleneck)
1.  **DOM Overhead:** ทุกครั้งที่เราใส่ Log ใหม่เข้าไป หรือ ขยับตัวเลข Browser ต้องคำนวณตำแหน่งภาพใหม่หมด (Reflow) ซึ่งหน่วงเครื่องมาก
2.  **Serial Blocking:** การคุยกับบอร์ดผ่าน Android Bridge อาจจะดึงความเร็ว CPU ของ WebView ไปครู่หนึ่ง
3.  **UI Complexity:** กราฟิกที่ซับซ้อนเกินไป (พวก Blur/Gradient) บน Tablet รุ่นเก่าจะทำเฟรมเรตตกครับ

## 2. 🛡️ แผนการแก้แบบถอนรากถอนโคน (Step-by-Step)

### ขั้นที่ 1: "Silent Mode" ระหว่างจ่ายน้ำ (LOG Decoupling)
*   **ปัจจุบัน:** แอปเขียน Log ลงหน้าจอตลอดเวลาที่จ่ายน้ำ (ทำให้จอต้องวาดภาพใหม่รัวๆ)
*   **ใหม่:** ในช่วงที่น้ำกำลังไหล เราจะแอบเก็บ Log ไว้ใน Memory แทน และ **ไม่แสดงผลบนหน้าจอเลย** ครับ จะโชว์เฉพาะยอดน้ำอย่างเดียว เพื่อให้ CPU ทุ่มเทให้ตัวเลขน้ำลื่นไหลที่สุด

### ขั้นที่ 2: กราฟิกแบบ "Android Lite"
*   **ปัจจุบัน:** มีเงา (Box-Shadow) และการไล่สี (Gradient) ที่ซับซ้อน
*   **ใหม่:** ลดการใช้ Shadow และ Blur ในจุดที่ไม่จำเป็น เพื่อให้ GPU ของ Tablet วาดภาพได้ 60fps จริงๆ

### ขั้นที่ 3: ระบบป้องการยอดค้าง (Deadman Switch)
*   ถ้าบอร์ดไม่ตอบกลับภายในวินาทีที่กำหนด เราจะไม่รอครับ เราจะให้หน้าจอรันต่อไปด้วยความพริ้ว และค่อย Sync ข้อมูลย้อนหลังเพื่อความเสถียร
